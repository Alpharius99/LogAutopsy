<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Extension Host runtime | Extension code runs in VS Code's extension host process with full Node.js access |
| Webview sandbox | Webview runs in an iframe with CSP restrictions — separate from extension host |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03 | Spoofing | Webview content | accept | Phase 1 webview has no scripts and no external resources; CSP `default-src 'none'` blocks all loading. No user input is accepted. |
| T-01-04 | Tampering | Webview CSP | mitigate | `enableScripts: false` + CSP `default-src 'none'; style-src 'unsafe-inline'` prevents any script execution. Phase 3 will add nonce-based script-src when scripts are needed. |
| T-01-05 | Information Disclosure | Extension host | accept | Phase 1 extension is read-only stubs; no file system access, no network calls, no secrets handled. |
| T-01-06 | Elevation of Privilege | src/core/ boundary | mitigate | D-02 enforces that src/core/ has zero vscode imports — core logic cannot access VS Code APIs or workspace. Verified by grep in Task 1. |
</threat_model>
