<!-- Step marker exact format — verified from real log line 484 -->
Step-start marker (GherkinExecutor\ExecuteStep:187):
"Next test step 'StartCarBusTrace' (location '9:4', keyword 'When ', keyword type 'Action', argument '')."

location format: 'col:line' — SECOND number is the 1-based feature file line.
'9:4' → featureLine=4.
Use featureLine to look up StepPhaseMap: stepPhaseMap.get(featureLine).

Step-result marker (GherkinExecutor\ExecuteStep:221):
"Result of test step 'StartCarBusTrace'."

KeywordTranslator failure markers:
"The test action 'X' is failed."     → KW_ACTION_FAILED
"The test check 'X' is failed."      → KW_CHECK_FAILED

<!-- Step marker regexes from RESEARCH.md Pattern 3 -->
```typescript
const STEP_START = /GherkinExecutor\\ExecuteStep:187 - Next test step '(.+?)' \(location '(\d+):(\d+)', keyword '(.+?)', keyword type '(.+?)', argument '(.*?)'\)\./;

```

<!-- endLine boundary rule — from RESEARCH.md Pitfall 6 -->
For step N: endLine = step(N+1).startLine - 1
For the last step: endLine = totalLines (last event's fileLineNumber)
Range is inclusive: startLine <= fileLineNumber <= endLine

import * as assert from 'assert';
import * as path from 'path';

Implement extractSteps(events: LogEvent[], stepPhaseMap: StepPhaseMap): StepContext[] following
PATTERNS.md skeleton (lines 306–337):

Key rules:
1. totalLines = events at last position's fileLineNumber (or 0 if empty)
2. For each event, run STEP_START.exec(event.message)
3. If match: m[3] = featureLine (second number in 'col:line' format — CRITICAL: m[2]=col, m[3]=line)
4. stepPhaseMap.get(featureLine) — if undefined, skip this step (unknown step)
5. Close current step: current.endLine = event.fileLineNumber - 1
6. Open new step: startLine = event.fileLineNumber, endLine = totalLines (overwritten by next match)
7. failedByKeywordTranslator starts false; set to true if KW_ACTION_FAILED or KW_CHECK_FAILED fires
8. After loop: if current, push to contexts
9. Return contexts

The location in the created StepContext.step.location should come from the StepPhaseMap entry
(which was built by featureParser using the AST location). Do not override it with the log marker's
location string.
